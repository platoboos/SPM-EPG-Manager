# Checkliste vor einer Veroeffentlichung

## Dateien

- keine `.env` Datei hochladen
- keine Passwoerter in Beispielen stehen lassen
- keine privaten EasyEPG `settings.json` hochladen
- keine Reports mit persoenlichen Portaldaten veroeffentlichen

## EasyEPG

- nur bereinigte Vorlagen teilen
- Sessions und Cookies entfernen
- API-Keys entfernen
- private XMLTV-URLs pruefen

## GitHub

- README auf den aktuellen Einzeiler pruefen
- Repository-Name und Links pruefen
- Screenshot nur ohne Zugangsdaten veroeffentlichen
- vor einem Release einen Dry-Run auf einem Testsystem machen

## Forum

- Voraussetzungen nennen
- klar schreiben: Erst pruefen, dann speichern
- GitHub-Link und Installations-Einzeiler angeben
- bei Supportfragen nach SPM-Version, Sendername und Dry-Run-Report-Ausschnitt fragen

## Spaeter sinnvoll

- erste GitHub-Release-Version erstellen
- Changelog fuer groessere Alias-Updates fuehren
