# Checkliste vor einer Veröffentlichung

## Dateien

- keine `.env` Datei hochladen
- keine Passwörter in Beispielen stehen lassen
- keine privaten EasyEPG `settings.json` hochladen
- keine Reports mit persönlichen Portaldaten veröffentlichen

## EasyEPG

- nur bereinigte Vorlagen teilen
- Sessions und Cookies entfernen
- API-Keys entfernen
- private XMLTV-URLs prüfen

## GitHub

- README auf den aktuellen Einzeiler prüfen
- Repository-Name und Links prüfen
- Screenshot nur ohne Zugangsdaten veröffentlichen
- vor einem Release einen Dry-Run auf einem Testsystem machen

## Forum

- Voraussetzungen nennen
- klar schreiben: Erst prüfen, dann speichern
- GitHub-Link und Installations-Einzeiler angeben
- bei Supportfragen nach SPM-Version, Sendername und Dry-Run-Report-Ausschnitt fragen

## Später sinnvoll

- erste GitHub-Release-Version erstellen
- Changelog für größere Alias-Updates führen
